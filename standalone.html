一、隨選氣象資料後端解決方案

蒐集氣象局 10 種以上的 Opendata，透過 RESTful API 供您隨選資料集。


二、設計與開發帳號與權限系統

此服務是氣象局監控系統的一個模組。工作內容包含MySQL 資料庫設計、建立資料模型、Django RestFramework WebAPI、提供JS SDK給前端、串接Oauth2、Gitlab CI/CD的實現。


三、主導開發氣象局監控系統

專為氣象局打造的一站式平台，用於觀察Infra、apps、files來幫助IT的有效管理。我去年做為工程師，從系統設計到開發與測試；今年作為窗口/PM，與各部門客戶、不同的開發團隊做溝通；作為維運人員，在各種環境裸機上搭建平台、幫助開發人員、整合各模組的功能。


四、氣象局西北太平洋資料庫 API management 開發

氣象局面向學術單位，提供API下載海象資料，希望有一套保護API 的機制。我提供的Solution是導入Kong，建立API management 保護層。工作內容包括Client驗證、授權（資料黑名單/Client白名單檢查）、存取頻率限制、加密URI檔案路勁、timeout調校，為了和既有的上游APIs 兼容，需要自行寫 Kong 的 Plugins。


五、後端ETL程式開發

採用Pipeline架構風格，管道的起點有三種方式：時間驅動、檔案到位事件、Webhook。主要是利用petl、pandas、xpath等套件，對XML檔案執行Extract、transform、load等等以管道串聯的篩選器任務，最後把資料寫入 DB。


六、產品產製程式開發

由於氣象產品的時間 Windows Size 經常需要調整，而且有很 多資料點位對應的運算。為了更有彈性、更具效能，採用微核 心架構風格，由核心從 MySQL 讀取註冊表，調用註冊表內的 模組並注入參數。


七、氣象局跨領域應用 App 後端開發

有健康氣象、休閒旅遊、校園氣象以及漁業氣象 App。 開發內容以 django 為主。工作內容包含旅遊景點搜尋、規劃行 程，各種氣象參數的警示；各種氣象主題的訂閱與推播，如：熱 傷害、豪大雨、颱風、寒流等的訂閱與推播 處理。
